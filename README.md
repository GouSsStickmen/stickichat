# StickiChat

Швидкий десктопний Twitch-чат клієнт для модераторів — мультиакаунт, мультичат, повна підтримка емоутів 7TV / BTTV / FFZ / Twitch.

## Можливості

- **Мод-режим**: налаштовувані кнопки (таймаути, бан, розбан, видалення повідомлень, warn, шатаут, швидкий рейд, анонси, сніпети, швидкі посилання) — на ховері повідомлення та в тулбарі каналу
- **Видалення власних повідомлень** (якщо ти модератор) — повідомлення отримують справжні server id
- **Мультиакаунт**: скільки завгодно акаунтів, окремий акаунт-відправник у кожній панелі (одночасне використання)
- **Мультичат**: вкладки + сітка панелей (1–4 колонки або авто), скільки завгодно чатів
- **Емоути**: Twitch, 7TV (включно з zero-width), BTTV, FFZ — глобальні та канальні, анімовані, автокомпліт по Tab
- **Карточка користувача**: вік акаунта, останні повідомлення в каналі, швидкі мод-дії
- **Історія чату** при відкритті каналу (recent-messages.robotty.de, як у Chatterino)
- Українська та англійська мови, темна/світла тема

> **Пін повідомлень**: Twitch поки не має публічного API для закріплення повідомлень (у них це "заплановано"). Кнопка є в тулбарі як заглушка; замість піна використовуй анонси.

## Запуск (розробка)

```
npm install
npm run dev
```

## Збірка інсталятора (Windows)

```
npm run dist
```

Інсталятор з'явиться в папці `release/`.

## Перший запуск

1. Створи безкоштовний застосунок на [dev.twitch.tv/console](https://dev.twitch.tv/console/apps/create):
   - Name — будь-яка, наприклад `StickiChat`
   - OAuth Redirect URL — `http://localhost`
   - Category — `Chat Bot`
   - **Client Type — `Public`** (обов'язково, інакше авторизація не працюватиме)
2. Скопіюй Client ID і встав в онбординг-екран
3. Додай акаунт(и) через код на twitch.tv/activate

Токени зберігаються зашифрованими (Windows DPAPI через Electron safeStorage).

## Роздати застосунок іншим людям

Client ID для Public-клієнта не є секретом — саме так задумано в OAuth2, і саме так працює Chatterino та інші подібні клієнти. Він лише каже Twitch "цей запит на авторизацію йде від StickiChat"; доступу до чужих акаунтів він не дає — кожен користувач все одно проходить свій власний Device Code Flow і отримує власний токен.

Щоб друзям не доводилось реєструвати застосунок самим:

1. Зареєструй застосунок один раз (крок вище) і встав свій Client ID у [src/renderer/src/config/defaultClientId.ts](src/renderer/src/config/defaultClientId.ts)
2. Збери інсталятор: `npm run dist`
3. Роздай `release/StickiChat Setup.exe` — онбординг у друзів одразу почнеться з кроку "додати акаунт"

Інсталятор не підписаний сертифікатом, тому Windows SmartScreen покаже попередження при першому запуску — це очікувано, треба натиснути "Докладніше" → "Виконати попри це".

## Оновлення застосунку (авто-оновлення через GitHub Releases)

Додаток сам перевіряє оновлення при старті (і раз на 2 години) та пропонує "Перезапустити й встановити" — і собі, і всім, кому ти його роздав. Разова настройка:

1. Створи **безкоштовний GitHub-репозиторій** (публічний або приватний) — наприклад `stickichat`
2. Встав своє ім'я користувача GitHub у `owner` в [electron-builder.yml](electron-builder.yml) (заміни `REPLACE_WITH_GITHUB_USERNAME`), і за потреби зміни `repo`
3. Створи **Personal Access Token**: GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → права **`repo`** (для приватного репо) або **`public_repo`** (для публічного)
4. Перед релізом постав токен у змінну середовища (в PowerShell): `$env:GH_TOKEN = "твій_токен"`
5. Підніми версію в [package.json](package.json) (`"version": "0.2.0"` тощо — кожен реліз має мати нову вищу версію!)
6. Запусти `npm run release` — це збере інсталятор **і автоматично завантажить його на GitHub Releases** разом зі службовим файлом `latest.yml`, який і читають запущені копії застосунку, щоб дізнатися про нову версію

Після цього всі версії StickiChat, які вже стоять у тебе й у друзів, самі побачать банер оновлення й запропонують встановити нову — переустановлювати вручну більше не треба.

`npm run dist` (без `--publish`) лишається для локальної збірки без завантаження на GitHub — зручно для тестування інсталятора самому.

## Архітектура (коротко)

- Читання чату — одне анонімне IRC WebSocket-з'єднання на всі канали (єдине джерело повідомлень, у всіх повідомлень справжні id)
- Надсилання — окреме IRC-з'єднання на кожен акаунт
- Усі модераційні дії — Twitch Helix API
- Авторизація — OAuth Device Code Flow (public client, авто-refresh токенів)
